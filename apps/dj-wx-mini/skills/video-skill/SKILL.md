# video-skill 无水印视频解析（小微 AI Handoff 模式）

> 本 skill 适配微信小微 AI Handoff：对话内只展示**文本回复 + 小程序卡片**，
> 用户点击卡片后 **handoff 接力**进入小程序的解析结果页，在小程序内完成查看/保存/下载等操作。

## 业务流程图

```
用户意图
  │
  ├─ 粘贴分享链接/文案 ──→ parseVideo ──→ 文本 + 小程序卡片
  │    （抖音/B站/快手/小红书/微博/视频号/TikTok）    点击卡片 → 接力页 video（查看/保存/下载）
  │
  └─ 其他意图（超出能力范围）──→ 文本回复"暂不支持"
```

## 原子接口 ↔ 接力页映射

| 接口 | 作用 | handoff.query | 接力页 pagePath |
|------|------|---------------|-----------------|
| parseVideo | 解析分享链接，提取无水印视频/图片 | `url=xxx&platform=douyin` | /pages/video/video |

## 业务约束

### 0. 能力边界（最高优先级）
- 本 skill 只有一个接口：`parseVideo`。它负责把用户给的分享链接解析成无水印视频/图片。
- **当用户意图超出解析范围时（如：生成视频、剪辑、AI对话、查询账号信息等），必须直接回复"该能力暂不支持"，禁止调用 parseVideo。**

### 1. 输出形态
- 对话内**只输出文本 + 小程序卡片**，禁止以纯文本列出视频地址或图片链接。
- 解析成功后，用一句简短话术引导用户点击小程序卡片进入结果页。

### 2. 调用规则
- `url` 必须来自用户原话，禁止编造链接。
- 用户没给链接时，先追问用户索要，不要调用 parseVideo。
- 解析失败时，告知用户可能的原因，禁止用相同参数反复重试。

### 3. 进小程序后的后续
- 保存视频、下载、复制文案等操作**均在小程序接力页内完成**，不再回到对话调用接口。

## 用户意图分流

- 粘贴了分享链接/文案 → `parseVideo`
- 说"帮我解析/下载这个视频" + 给了链接 → `parseVideo`
- 说"帮我解析这个视频"但没给链接 → 先追问链接
- 要识别文字/OCR → 暂不支持（后续可扩展）
- 要写文案 → 暂不支持（后续可扩展）
- 要去除图片水印 → 暂不支持（后续可扩展）
- 其他无关意图 → 正常文本回复，不调用接口
